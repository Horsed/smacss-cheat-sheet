<h1>cheat sheet for smacss with sass and bem</h1> based on [this article](https://medium.com/objects-in-space/f6f404727) by [Andrew Colclough](https://twitter.com/wtc) (which refers [these other links](#sources)) <h2>pattern</h2> ```html <div class="base-module--specific-module"> <div class="specific-module__child-module"> <div class="child-module">...</div> <div class="child-module is-active">...</div> <div class="child-module--is-last">...</div> </div> </div> ``` * modules define their style and the positioning of their child components * a module name contains it's base module's name * global states are defined globally * module specific states are defined inside the module <h2>directory structure</h2> <pre> css/ |-- base/ | |-- fonts.scss | |-- colors.scss | |-- mixins.scss | |-- variables.scss |-- states/ | |-- <a href="#states.scss" class="link">states.scss</a> |-- modules/ | |-- <a href="#base-module.scss" class="link">base-module.scss</a> | |-- <a href="#child-module.scss" class="link">child-module.scss</a> </pre> <h2 id="states.scss">states.scss</h2> ```css .is-active { ... } /*global state*/ ``` <h2 id="base-module.scss">base-module.scss</h2> ```css .base-module { ... } /*style of a module*/ .specific-module { @extend .parent; ... } /*style of a specific module NEVER @extend outside of module*/ .specific-module__child-module { ... } /*positioning of child module*/ ``` <h2 id="child-module.scss">child-module.scss</h2> ```css .child-module { ... } /*style of a module*/ .child-module--is-last { ... } /*module specific state*/ ``` <h2 id="sources">sources</h2> * http://smacss.com/ * http://bem.info/ * http://webuild.envato.com/blog/how-to-scale-and-maintain-legacy-css-with-sass-and-smacss/ * https://medium.com/objects-in-space/f6f404727
